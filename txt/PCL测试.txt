function exportLocalStorageAndToken() {
  var script = [];
  // 遍历localStorage并生成对应的setItem语句
  for (var i = 0; i < localStorage.length; i++) {
    var key = localStorage.key(i);
    var value = localStorage.getItem(key);
    script.push(`localStorage.setItem('${key}', ${JSON.stringify(value)});`);
  }

  // 获取token并生成设置cookie的语句
  var token = localStorage.getItem("token");
  if (token) {

  console.log(script.join('\n'));
}

在 **Windows CMD** 中，你可以使用以下命令删除 `node_modules` 目录：  

---
rd /s /q node_module
### **方法 1：使用 `rd /s /q`**
```cmd
rd /s /q node_modules
```
✅ **解释：**  
- `rd`（`rmdir`）：删除目录的命令。  
- `/s`：删除目录及其所有子文件。  
- `/q`：静默删除，不提示确认。  

💡 **适用于：** 绝大多数情况下，快速删除 `node_modules` 目录。

---

### **方法 2：使用 `del` 和 `rmdir`**
```cmd
del /f /s /q node_modules
rmdir /s /q node_modules
```
✅ **适用于：** 如果 `rd` 命令报错时，可以尝试这个组合命令。

---

### **方法 3：使用 PowerShell**
如果 `cmd` 删除失败，可以使用 **PowerShell**：
```powershell
Remove-Item -Recurse -Force node_modules
```
✅ **适用于：** 某些情况下 `cmd` 无法删除 `node_modules`，PowerShell 更稳定。

---

### **方法 4：使用 `rimraf`（推荐）**
如果 `node_modules` 目录特别大，Windows 可能会因为 **路径过长** 导致删除失败，可以使用 `rimraf`：
```cmd
npx rimraf node_modules
```
✅ **适用于：**  
- `node_modules` 体积特别大，导致 Windows 无法直接删除。  
- **`rimraf`** 解决 Windows **路径过长** 问题。  
- **如果 `npx rimraf` 失败，可以全局安装**：
  ```cmd
  npm install -g rimraf
  rimraf node_modules
  ```

---

## **总结**
| 方法 | 适用情况 | 运行环境 |
|------|---------|---------|
| `rd /s /q node_modules` | **最常用，适用于大多数情况** | **CMD** |
| `del /f /s /q node_modules && rmdir /s /q node_modules` | CMD 删除失败时尝试 | CMD |
| `Remove-Item -Recurse -Force node_modules` | **PowerShell 方式，适用于删除失败的情况** | **PowerShell** |
| `npx rimraf node_modules` | **路径过长或删除失败** | **CMD / PowerShell** |

如果你在 **Vue 项目** 运行 `npm install` 时遇到问题，建议先删除 `node_modules`，然后重新安装：
```cmd
rd /s /q node_modules
del package-lock.json
npm install
```
这样可以避免 `npm install` 失败的情况。🚀



在使用 `adb logcat` 查看日志时，出现汉字乱码的情况通常是由于编码问题引起的。解决方案如下：

### 解决方案 1：指定输出编码
可以通过设置命令行的输出编码来解决汉字乱码问题。

#### Windows
1. 打开命令行窗口（CMD）。
2. 执行以下命令将编码设置为 UTF-8：
   ```bash
   chcp 65001
   ```
3. 重新运行 `adb logcat` 命令，日志中应该能正确显示汉字。

#### Mac/Linux
Mac 或 Linux 系统通常默认支持 UTF-8 编码，但如果还是出现乱码，可以尝试通过以下方式设置：

```bash
export LC_ALL=en_US.UTF-8
adb logcat
```

### 解决方案 2：使用第三方工具
可以使用一些日志查看工具来解析 `adb logcat` 输出，常见工具有：
- **Android Studio 的 Logcat**：直接在 Android Studio 内部查看 logcat，通常不会出现乱码问题。
- **Pidcat**：Pidcat 是一个第三方工具，可以很好地处理 `adb logcat` 输出中的中文问题。

### 解决方案 3：保存日志后转换编码
1. 将日志保存到文件：
   ```bash
   adb logcat -v time > log.txt
   ```
2. 使用文本编辑器（如 Notepad++）打开 `log.txt`，并将文件编码转换为 UTF-8。

这样应该可以解决 `adb logcat` 查看日志时的汉字乱码问题。

要将 `div` 里面的内容上下居中显示，可以使用以下几种方法之一：

### 方法一：使用 Flexbox
这是最常用且最灵活的方法。

```css
.container {
  display: flex;
  align-items: center;
  justify-content: center;
  height: 100vh; /* 或者设置为容器的高度 */
}
```

### 方法二：使用 Grid 布局
Grid 布局也可以轻松实现上下居中。

```css
.container {
  display: grid;
  place-items: center;
  height: 100vh; /* 或者设置为容器的高度 */
}
```

### 方法三：使用 `line-height`
如果你的内容只有一行文本，可以使用 `line-height`。这种方法不适用于多行文本或复杂内容。

```css
.container {
  height: 200px; /* 容器的高度 */
  line-height: 200px; /* 和容器的高度一致 */
  text-align: center;
}
```

### 方法四：使用绝对定位
这种方法适用于所有类型的内容，但需要知道容器的高度。

```css
.container {
  position: relative;
  height: 200px; /* 容器的高度 */
}

.container .content {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
```

要在 CSS 中设置 `div` 里面的内容超出部分显示省略号，可以使用以下代码：

### 单行文本省略号
```css
div {
  white-space: nowrap;        /* 防止文本换行 */
  overflow: hidden;           /* 隐藏超出部分 */
  text-overflow: ellipsis;    /* 超出部分显示省略号 */
}
```

### 多行文本省略号
对于多行文本显示省略号，你可以使用以下方法：

```css
div {
  display: -webkit-box;        /* 将元素设置为弹性盒子模型 */
  -webkit-box-orient: vertical; /* 设置为垂直排列 */
  -webkit-line-clamp: 3;        /* 限制显示的行数，例如3行 */
  overflow: hidden;             /* 隐藏超出部分 */
}
```

### 解释：
- **单行文本：** 使用 `white-space: nowrap;` 防止文本换行，`overflow: hidden;` 隐藏超出部分，`text-overflow: ellipsis;` 显示省略号。
- **多行文本：** 使用 `-webkit-line-clamp` 来指定显示的行数，并配合 `-webkit-box` 实现多行省略号效果。

你可以根据具体需求选择单行或多行省略号样式。
